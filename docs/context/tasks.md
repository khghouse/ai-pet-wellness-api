# 작업 목록

> 실제로 진행하기로 결정한 작업을 관리하는 곳.
> 후보 이슈는 `backlog.md`, 완료/검증 이력은 `progress.md`, 설계 결정은 `docs/adr/`를 기준으로 한다.

---

## 작성 규칙

`tasks.md`는 실행 목록만 관리한다.
작업 후보는 `backlog.md`에서 검토한 뒤 이 파일로 옮기고, 옮긴 항목은 `backlog.md`에서 제거한다.

### 작업 작성 형식

```md
- [ ] 작업명
  - **목표:** ...
  - **배경:** ...
  - **완료 기준:**
    - [ ] ...
  - **검증:** `검증 명령`
  - **참고:**
    - ...
```

### 상태 관리 기준

- `진행 중`에는 현재 집중할 작업만 둔다.
- `대기 중`에는 가까운 시일 내 진행하기로 결정한 작업을 둔다.
- 완료한 작업은 `완료`로 옮기고, 검증 결과는 `progress.md`에도 기록한다.
- 진행하지 않기로 한 작업은 삭제하거나 `backlog.md`로 되돌린다.

---

## 진행 중

> 현재 집중 중인 작업. 동시에 여러 개를 넣지 않는다.


---

## 대기 중

> backlog에서 승격된 실행 예정 작업.


---

## 완료

> 완료된 작업의 짧은 기록. 상세 변경과 검증 이력은 `progress.md`에 기록한다.
