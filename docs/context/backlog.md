## P1 — 높음

- **회원 탈퇴 시 인증 토큰 즉시 폐기**
  - **출처:** AI
  - **문제:** 회원 탈퇴 후 로그인과 재발급, 내 정보 조회는 차단되지만 기존 Access Token은 만료 전까지 JWT 인증 자체는 통과할 수 있다.
  - **제안:** 회원 탈퇴 시 Refresh Token을 삭제하고 현재 Access Token을 Redis 블랙리스트에 추가한다.
  - **영향:** 향후 보호된 도메인 API가 추가돼도 탈퇴 회원의 기존 토큰 사용을 즉시 차단할 수 있다.
  - **트레이드오프:** 회원 탈퇴 흐름이 인증 모듈의 토큰 저장소와 협력해야 한다.

## P2 — 보통

- **리워드 수령 자격 검증 설계 및 구현**
  - **출처:** 사용자
  - **문제:** 반려견 검증 없이 미션에 참여할 수 있어도, 경제적 가치가 있는 리워드는 실제 사용자와 반려견을 확인해야 한다.
  - **제안:** 미션 참여와 달성은 개방하되, 리워드 수령 시 사용자 본인인증과 반려견 검증 완료를 모두 요구한다.
  - **영향:** 리워드 부정 수급을 줄이고, 가족 구성원이 리워드를 수령할 때도 사용자와 반려견 검증을 분리해 판단할 수 있다.
  - **트레이드오프:** 본인인증 및 반려견 검증 외부 연동, 개인정보 처리와 검증 실패 정책이 필요하다.
