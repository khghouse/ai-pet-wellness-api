## P1 — 높음

- **회원 탈퇴 시 인증 토큰 즉시 폐기**
  - **출처:** AI
  - **문제:** 회원 탈퇴 후 로그인과 재발급, 내 정보 조회는 차단되지만 기존 Access Token은 만료 전까지 JWT 인증 자체는 통과할 수 있다.
  - **제안:** 회원 탈퇴 시 Refresh Token을 삭제하고 현재 Access Token을 Redis 블랙리스트에 추가한다.
  - **영향:** 향후 보호된 도메인 API가 추가돼도 탈퇴 회원의 기존 토큰 사용을 즉시 차단할 수 있다.
  - **트레이드오프:** 회원 탈퇴 흐름이 인증 모듈의 토큰 저장소와 협력해야 한다.

## P2 — 보통

- **common-modules 안정 버전 전환**
  - **출처:** AI
  - **문제:** 현재 연동한 `0.1.0-SNAPSHOT`은 동일한 버전 좌표의 artifact가 변경될 수 있어 재현 가능한 빌드를 보장하기 어렵다.
  - **제안:** `common-modules`의 안정 버전이 배포되면 `common-web`, `common-logging`, `common-auth` 의존성을 해당 버전으로 전환한다.
  - **영향:** 로컬과 CI에서 동일한 artifact를 사용해 빌드 재현성을 높일 수 있다.
  - **트레이드오프:** 공통 모듈의 릴리스와 버전 변경을 별도로 관리해야 한다.
  - **참고:**
    - https://github.com/khghouse/common-modules
