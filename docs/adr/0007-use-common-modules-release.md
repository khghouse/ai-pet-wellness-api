# 0007. common-modules 불변 릴리스 사용

## 상태

승인됨

## 배경

`common-modules:0.1.0-SNAPSHOT`은 같은 버전 좌표로 변경된 artifact를 다시 배포할 수 있어 Gradle 캐시 상태에 따라 서로 다른 구현이 사용될 수 있다.

JWT 고유 식별자 생성 문제를 수정한 `common-auth`도 동일한 SNAPSHOT 버전으로 재배포됐으므로 로컬과 CI에서 사용하는 artifact가 달라질 위험이 있다.

## 결정

- GitHub Packages에 배포된 첫 불변 릴리스 `0.1.0`을 사용한다.
- `common-web`, `common-logging`, `common-auth` 의존성을 `0.1.0`으로 고정한다.
- `common-core`는 공통 모듈의 전이 의존성으로 `0.1.0`을 사용한다.
- 공통 모듈을 변경할 때는 기존 버전을 재배포하지 않고 새 버전을 발행한다.

## 결과

- 로컬과 CI가 동일한 공통 모듈 artifact를 사용해 빌드 재현성이 높아진다.
- 공통 모듈 변경 사항을 사용하려면 새 버전을 발행하고 소비 프로젝트의 의존성을 명시적으로 갱신해야 한다.
- GitHub Packages를 읽기 위한 인증 정보와 권한은 계속 필요하다.

## 검토한 대안

- `0.1.0-SNAPSHOT` 유지:
  - 이유: 변경 배포는 빠르지만 캐시 상태에 따라 서로 다른 artifact를 사용할 수 있으므로 채택하지 않는다.
- 변경 의존성 캐시 시간을 0으로 설정:
  - 이유: 최신 SNAPSHOT을 받을 수 있지만 빌드 재현성을 보장하지 못하고 매 빌드마다 원격 저장소를 확인하므로 채택하지 않는다.
