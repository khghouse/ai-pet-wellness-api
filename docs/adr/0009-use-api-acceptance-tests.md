# 0009. API 인수 시나리오 테스트 도입

## 상태

승인됨

## 배경

Controller, Service, MockMvc 기반 통합 테스트는 개별 API와 애플리케이션 계층의 동작을 검증한다. 그러나 신규 사용자가 회원 가입부터 로그인, 인증된 반려견 등록까지 완료하는 핵심 흐름을 실제 HTTP 서버 환경에서 검증할 필요가 있다.

## 결정

- 핵심 사용자 흐름은 API 인수 시나리오 테스트로 자동화한다.
- API 인수 시나리오 테스트는 `@SpringBootTest(webEnvironment = RANDOM_PORT)`로 실행한다.
- 테스트는 실제 HTTP 요청을 사용하고, Redis 같은 외부 인프라는 Testcontainers로 격리한다.
- 시나리오 정의는 `docs/test-scenarios/SCN-xxx-*.md`, 실행 코드는 `src/test/java/.../acceptance/`에 둔다.
- 초기 범위는 회원 가입부터 반려견 등록까지의 SCN-001로 제한한다.

## 결과

- 핵심 사용자 흐름의 회귀를 CI에서 자동으로 감지할 수 있다.
- 실제 HTTP 서버와 인증 필터, DB, Redis를 함께 검증한다.
- 테스트 실행 시간이 늘어나므로, 상세 분기 검증은 하위 테스트 계층에 유지하고 인수 시나리오는 소수의 핵심 흐름으로 관리한다.

## 검토한 대안

- 기존 MockMvc 기반 통합 테스트만 사용:
  - 이유: 빠르고 현재 API 계층을 검증할 수 있지만 실제 HTTP 서버 실행과 사용자 여정 검증 범위가 부족해 채택하지 않는다.
- 배포된 외부 테스트 환경에 직접 요청:
  - 이유: 운영 환경과 더 유사하지만 환경 의존성과 실행 비용이 커서 CI 기본 검증으로 채택하지 않는다.
