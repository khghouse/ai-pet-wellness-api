# 0008. 반려동물 도메인 모델 설계

## 상태

승인됨

## 배경

반려견은 여러 회원이 가족 관계로 함께 관리할 수 있고, 산책, 미션, 리워드 기능은 반려견과 실제 수행 회원을 함께 식별해야 한다.

또한 펫 웰니스 서비스에서 체중 변화는 유의미한 건강 정보이므로 현재 체중만 저장하면 변화 추이를 제공할 수 없다.

## 결정

- 회원과 반려견의 관계는 직접 연결하지 않고 `PetMembership` 연관 엔티티로 관리한다.
- `PetMembership`은 `OWNER`, `FAMILY` 역할과 `ACTIVE`, `LEFT` 상태를 가진다.
- 반려견 등록 시 등록 회원의 `OWNER`, `ACTIVE` 멤버십을 함께 생성한다.
- 견종은 `Breed` 기준 데이터로 관리하고, 신규 등록 가능 여부는 `active`로 관리한다.
- 반려견 체중은 `PetWeight` 추가 전용 이력으로 관리한다.
- 현재 체중은 가장 최근의 `PetWeight` 이력으로 조회하며, `Pet`에 중복 저장하지 않는다.
- 반려견 검증은 등록과 분리된 후속 요구사항으로 관리한다.

## 결과

- 가족 공유와 산책 수행자, 리워드 수령자를 구분할 수 있다.
- 한 반려견의 체중 변화를 시간순으로 조회할 수 있다.
- 현재 체중 조회에는 최신 이력 조회가 필요하지만, 데이터 중복과 동기화 문제를 피할 수 있다.
- 견종을 기준 데이터로 관리해 직접 입력으로 인한 표기 불일치를 줄일 수 있다.
- 반려견 검증과 리워드 지급 정책은 외부 연동과 개인정보 정책을 검토한 뒤 별도 기능으로 구현해야 한다.

## 검토한 대안

- 회원과 반려견 직접 연관관계:
  - 이유: 단순하지만 가족 공유와 관계별 역할, 산책 수행자 식별을 표현하기 어려워 채택하지 않는다.
- `Pet`에 최신 체중만 저장:
  - 이유: 조회는 단순하지만 체중 변화 추이를 제공할 수 없어 채택하지 않는다.
- 견종을 텍스트로 직접 입력:
  - 이유: 초기 구현은 빠르지만 표기 불일치와 이후 견종별 기능 확장이 어려워 채택하지 않는다.
