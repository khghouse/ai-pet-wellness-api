# REQ-006: 반려견 등록 및 소유자 멤버십 생성

- **목표:** 인증된 회원은 반려견을 등록하고, 해당 반려견의 소유자 멤버십을 생성할 수 있다.
- **행위자:** JWT 인증된 회원
- **범위:**
  - 반려견 등록
  - 활성 상태 견종 선택
  - 최초 체중 이력 생성
  - 등록 회원의 `OWNER` 멤버십 자동 생성
  - 견종 기준 데이터 초기 등록
- **관련 엔티티:** `Breed`, `Pet`, `PetWeight`, `PetMembership`
- **제외 범위:**
  - 반려견 정보 수정 및 삭제
  - 체중 이력 수정 및 삭제
  - 가족 구성원 초대 및 멤버십 수락
  - 반려견 검증
  - 산책, 미션, 리워드 기능
  - 견종 직접 입력
  - 견종 관리 기능

## API

- `POST /api/v1/pets`

## 입력값

- `name`: 반려견 이름
- `birthDate`: 생년월일, `yyyy-MM-dd` 형식
- `gender`: `MALE`, `FEMALE`, `UNKNOWN`
- `breedId`: 견종 식별자
- `weight`: 체중, kg 단위, 소수점 한 자리
- `neuteredStatus`: `NEUTERED`, `NOT_NEUTERED`, `UNKNOWN`

## 응답값

반려견 등록 응답은 `PetRegistrationResponse`로 제공한다.

- `id`: 반려견 식별자
- `name`: 반려견 이름
- `birthDate`: 생년월일
- `gender`: 성별
- `breed`: 견종 정보
  - `id`: 견종 식별자
  - `name`: 견종명
- `weight`: 등록 시 기록된 체중
- `neuteredStatus`: 중성화 상태
- `membershipRole`: 생성된 멤버십 역할, `OWNER`

## 정책

- JWT 인증된 회원만 반려견을 등록할 수 있다.
- 반려견 이름, 생년월일, 성별, 견종, 체중, 중성화 상태는 필수값이다.
- 생년월일은 미래 날짜일 수 없다.
- 견종은 활성 상태인 기준 데이터만 선택할 수 있다.
- 견종 직접 입력은 허용하지 않는다.
- 체중은 kg 단위의 양수이며, 소수점 한 자리까지만 입력할 수 있다.
- 반려견 등록 시 입력한 체중은 첫 번째 `PetWeight` 이력으로 저장한다.
- 첫 번째 체중 이력의 `measuredAt`과 `createdAt`은 반려견 등록 시각으로 기록한다.
- 반려견 등록과 첫 체중 이력, `OWNER` 멤버십 생성은 하나의 트랜잭션으로 처리한다.
- 생성되는 멤버십 역할은 `OWNER`, 상태는 `ACTIVE`다.
- 반려견은 등록 시 `deleted = false`로 저장한다.
- 반려견 등록만으로 반려견 검증이 완료되는 것은 아니다.
- 회원은 여러 반려견을 등록할 수 있다.

## 흐름

1. 인증된 회원이 반려견 정보를 입력한다.
2. 시스템은 필수값과 입력 형식을 검증한다.
3. 시스템은 생년월일이 미래가 아닌지 확인한다.
4. 시스템은 선택한 견종이 존재하고 활성 상태인지 확인한다.
5. 시스템은 반려견 정보를 저장한다.
6. 시스템은 등록한 체중을 첫 `PetWeight` 이력으로 저장한다.
7. 시스템은 등록 회원과 반려견을 연결하는 `OWNER`, `ACTIVE` 멤버십을 생성한다.
8. 시스템은 생성된 반려견 정보와 소유자 역할을 응답한다.

## 완료 기준

- [ ] 인증되지 않은 사용자는 반려견을 등록할 수 없다.
- [ ] 필수값이 누락되면 등록에 실패한다.
- [ ] 생년월일 형식이 올바르지 않거나 미래 날짜이면 등록에 실패한다.
- [ ] 성별 또는 중성화 상태 값이 유효하지 않으면 등록에 실패한다.
- [ ] 존재하지 않거나 비활성 상태인 견종을 선택하면 등록에 실패한다.
- [ ] 체중이 양수가 아니거나 소수점 한 자리를 초과하면 등록에 실패한다.
- [ ] 정상 요청이면 반려견이 생성된다.
- [ ] 정상 요청이면 첫 번째 체중 이력이 생성된다.
- [ ] 정상 요청이면 등록 회원의 `OWNER`, `ACTIVE` 멤버십이 함께 생성된다.
- [ ] 반려견, 체중 이력, 멤버십은 함께 성공하거나 함께 실패한다.
- [ ] 초기 견종 데이터가 Flyway를 통해 등록된다.

## 테스트 기준

- 정상 반려견 등록 성공
- 미인증 사용자 등록 실패
- 필수값 누락 실패
- 미래 생년월일 입력 실패
- 유효하지 않은 성별 또는 중성화 상태 입력 실패
- 존재하지 않는 견종 입력 실패
- 비활성 견종 입력 실패
- 유효하지 않은 체중 입력 실패
- 반려견 등록 시 첫 체중 이력 생성 확인
- 반려견 등록 시 `OWNER`, `ACTIVE` 멤버십 생성 확인
- 트랜잭션 실패 시 반려견, 체중 이력, 멤버십이 모두 저장되지 않는지 확인
