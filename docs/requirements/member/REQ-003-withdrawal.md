# REQ-003: 회원 탈퇴

- **목표:** 사용자는 자신의 회원 계정을 탈퇴할 수 있다.
- **행위자:** 사용자
- **범위:** JWT 인증 회원의 회원 탈퇴 처리
- **제외 범위:**
    - 비밀번호 재확인
    - 소셜 계정 연결 해제
    - 탈퇴 사유 저장
    - 탈퇴 취소
    - 탈퇴 회원 이메일 재사용 정책
    - 반려동물, 산책, 미션, 리워드 데이터 정리
    - 도메인 이벤트 발행

- **입력값:**
    - `Authorization`: `Bearer {accessToken}`

- **응답값:**
    - 없음

- **정책:**
    - `DELETE /api/v1/members/me`로 탈퇴를 요청한다.
    - JWT에서 추출한 회원 식별자를 사용한다.
    - 존재하는 회원만 탈퇴할 수 있다.
    - 이미 탈퇴한 회원은 다시 탈퇴할 수 없다.
    - 탈퇴 시 회원 상태는 `WITHDRAWN`으로 변경한다.
    - 탈퇴 시 삭제 여부는 `true`로 변경한다.
    - 탈퇴 시 삭제일시를 기록한다.
    - 탈퇴 후 해당 회원은 로그인할 수 없다.
    - 탈퇴한 회원의 이메일 재사용 가능 여부는 별도 요구사항에서 결정한다.
    - 현재 요구사항에서는 연관 도메인 데이터 정리를 수행하지 않는다.

- **흐름:**
    1. 사용자가 회원 탈퇴를 요청한다.
    2. 시스템은 JWT 인증 정보에서 회원 식별자를 추출한다.
    3. 시스템은 회원 식별자로 회원을 조회한다.
    4. 시스템은 회원이 존재하는지 확인한다.
    5. 시스템은 이미 탈퇴한 회원인지 확인한다.
    6. 시스템은 회원 상태를 `WITHDRAWN`으로 변경한다.
    7. 시스템은 삭제 여부를 `true`로 변경한다.
    8. 시스템은 삭제일시를 기록한다.
    9. 시스템은 성공 응답을 반환한다.

- **완료 기준:**
    - [ ] 존재하지 않는 회원이면 탈퇴에 실패한다.
    - [ ] 이미 탈퇴한 회원이면 탈퇴에 실패한다.
    - [ ] 정상 요청이면 회원 상태가 `WITHDRAWN`으로 변경된다.
    - [ ] 정상 요청이면 삭제 여부가 `true`로 변경된다.
    - [ ] 정상 요청이면 삭제일시가 기록된다.
    - [ ] 탈퇴한 회원은 로그인할 수 없다.
    - [ ] 탈퇴 성공 응답에는 별도 데이터가 포함되지 않는다.
