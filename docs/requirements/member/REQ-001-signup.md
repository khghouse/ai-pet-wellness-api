# REQ-001: 회원 가입

- **목표:** 사용자는 서비스 이용을 위해 회원 계정을 생성할 수 있다.
- **행위자:** 사용자
- **범위:** 이메일 기반 일반 회원 가입
- **관련 엔티티:** `Member`
- **제외 범위:**
  - 로그인
  - JWT 발급
  - 소셜 로그인
  - 이메일 인증
  - 비밀번호 재설정
  - 약관 동의
  - 탈퇴 회원 이메일 재사용 정책
- **입력값:**
  - 이메일
  - 비밀번호
- **응답값:**
  - `id`: 회원 식별자
  - `email`: 이메일
  - `status`: 회원 상태
- **정책:**
  - 이메일은 회원 식별자로 사용한다.
  - 동일한 이메일로 중복 가입할 수 없다.
  - 비밀번호는 평문으로 저장하지 않는다.
  - 비밀번호는 BCrypt 기반 단방향 해싱으로 저장한다.
  - 비밀번호 해싱에는 `org.springframework.security:spring-security-crypto`를 사용한다.
  - 비밀번호는 8자 이상 16자 이하로 입력한다.
  - 가입 시 회원 상태는 `ACTIVE`, 삭제 여부는 `false`로 저장한다.
  - 탈퇴한 회원의 이메일 재사용 가능 여부는 별도 요구사항에서 결정한다.
- **흐름:**
  1. 사용자가 이메일과 비밀번호를 입력한다.
  2. 시스템은 입력값 형식을 검증한다.
  3. 시스템은 이메일 중복 여부를 확인한다.
  4. 시스템은 비밀번호를 단방향 해싱한다.
  5. 시스템은 회원 정보를 저장한다.
  6. 시스템은 생성된 회원 정보를 응답한다.
- **완료 기준:**
  - [ ] 이메일, 비밀번호가 없으면 가입에 실패한다.
  - [ ] 이메일 형식이 올바르지 않으면 가입에 실패한다.
  - [ ] 비밀번호 정책을 만족하지 않으면 가입에 실패한다.
  - [ ] 이미 가입된 이메일이면 가입에 실패한다.
  - [ ] 정상 입력이면 회원이 생성된다.
  - [ ] 저장된 비밀번호는 평문이 아니다.
  - [ ] 가입 성공 응답에 비밀번호는 포함되지 않는다.
