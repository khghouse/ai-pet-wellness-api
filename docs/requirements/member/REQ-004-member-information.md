# REQ-004: 회원 정보 조회

- **상태:** REQ-005에 의해 대체됨
- **대체 내용:** 회원 식별자 입력 없이 JWT 인증 주체를 사용하는 `GET /api/v1/members/me`로 전환한다.
- **목표:** 사용자는 회원 식별자를 통해 자신의 회원 정보를 조회할 수 있다.
- **행위자:** 사용자
- **범위:** 회원 식별자 기반 단건 조회
- **제외 범위:**
    - JWT 기반 현재 로그인 회원 식별
    - 다른 회원 정보 조회 권한 제어
    - 회원 정보 수정
    - 비밀번호 조회 및 변경
    - 탈퇴 회원 정보 조회 정책

- **입력값:**
    - 회원 식별자 (`memberId`)

- **응답값:**
    - `id`: 회원 식별자
    - `email`: 로그인/회원 식별 이메일
    - `status`: 회원 상태

- **정책:**
    - 회원 식별자에 해당하는 회원이 존재해야 한다.
    - 탈퇴한 회원은 조회할 수 없다.
    - 비밀번호와 삭제 여부, 삭제 일시는 응답에 포함하지 않는다.
    - 현재는 `memberId`를 통해 조회하며, JWT 인증 도입 시 인증 정보에서 회원 식별자를 추출하도록 변경한다.

- **흐름:**
    1. 사용자가 회원 식별자를 전달한다.
    2. 시스템은 회원 존재 여부를 확인한다.
    3. 시스템은 탈퇴 회원 여부를 확인한다.
    4. 시스템은 회원 정보를 응답한다.

- **완료 기준:**
    - [ ] 존재하는 활성 회원을 조회할 수 있다.
    - [ ] 존재하지 않는 회원을 조회하면 실패한다.
    - [ ] 탈퇴한 회원을 조회하면 실패한다.
    - [ ] 응답에 비밀번호가 포함되지 않는다.
    - [ ] 응답에 삭제 여부와 삭제 일시가 포함되지 않는다.

- **예외:**
    - 존재하지 않는 회원
    - 탈퇴한 회원

- **테스트 기준:**
    - 정상 회원 조회 성공
    - 존재하지 않는 회원 조회 실패
    - 탈퇴 회원 조회 실패
    - 응답에 민감 정보가 포함되지 않는지 확인
